## 📋 ملخص المشروع
مشروع لوحة تحكم إدارية لنظام مدرستي (Madrasti) – نظام SaaS لإدارة المدارس المتعددة. مبنية على **Next.js 14** مع **App Router**، تدعم اللغة العربية فقط (RTL)، وتتواصل مع API خارجي.

**الهدف**: تحويل المشروع من حالة تطويرية إلى منصة إنتاجية متكاملة.

## 3. جميع الميزات (Features)

### 3.1 الصفحة الرئيسية `/admin`
- بطاقات إحصائية ثابتة (حضور، نقاط، مراسلات، حفظ)
- إجراءات سريعة (تسجيل طالب، مراسلة، غياب، تقرير)
- جدول الطلاب المعرضين للخطر (نسبة الغياب، نقاط السلوك)

- مصفوفة صلاحيات لكل الموارد (7 موارد رئيسية)

- عرض المستخدم الحالي مع أدواره

### 3.3 الحضور `/admin/attendance`
- ماسح QR محاكى (إدخال يدوي لرمز الطالب)

- إحصائيات الحضور (إجمالي، حاضر، غائب، متأخر)

- إعادة تعيين الجلسة

### 3.4 المراسلات `/admin/correspondence`
- إحصائيات (إجمالي، وارد، صادر، عاجل)
- فلترة (الاتجاه، الأولوية، الحالة، بحث)
- فرز (التاريخ، الأولوية، الرقم المرجعي)

- لوحة التفاصيل الجانبية
- إدارة المرفقات (رفع، تنزيل، قائمة)
- إنشاء رد مرتبط
- تغيير الحالة

### 3.5 الحلقات `/admin/halaqat`
- بطاقات الحلقات (الاسم، المستوى، المعلم، عدد الطلاب)
- قائمة الطلاب المتاحين لإضافتهم/إزالتهم
- تحديد هدف الحفظ اليومي (سورة، آية)
- تقدم الطلاب (عدد الآيات والأجزاء المحفوظة)

### 3.6 نقاط السلوك `/admin/points`
- نموذج تسجيل نقاط (إيجابي/سلبي)
- بطاقات ملخص (إيجابي، سلبي، صافي، عدد العمليات)
- سجل قابل للفلترة (الكل/إيجابي/سلبي)
- تحقق من صحة الإدخال

### 3.7 التقارير `/admin/reports`
- **تقرير تقدم الحلقات**: إحصائيات، أشرطة تقدم، توزيع المستويات
- **لوحة متصدرى الشريعة**: ترتيب الطلاب، فلترة بالمستوى

- **لوحة المخاطر السلوكية**: منزلق العتبة، جدول الطلاب المعرضين للخطر

## 4. تدفق البيانات (Data Flow)

```
المكون/الصفحة
    ↓
services/api/*.ts  ← دوال API مصنفة
    ↓
apiInterceptor.ts  ← حقن Bearer Token + X-School-ID
    ↓
apiClient.ts       ← HTTP client (fetch-based)
    ↓                    ├── إعادة المحاولة (retry)
    │                    ├── مهلة زمنية (timeout)
    │                    ├── Idempotency-Key للـ POST/PATCH
    │                    ├── X-Request-ID تتبع
    │                    └── 401 → refresh token تلقائي
    ↓
envelopeHandler.ts  ← تطبيع الأخطاء واستخراج pagination
    ↓
[API] → https://api.madrasti.xn--mgbaab0cxheq.tech/api/v1
```

### 4.1 ApiClientInstance
- `get<T>`, `post<T>`, `put<T>`, `patch<T>`, `delete<T>`
- `request<T>(method, url, body?, options?)` — الطريقة الأساسية
- `setBaseUrl()`, `setHeader()`, `removeHeader()`, `getConfig()`

### 4.2 ApiResponse<T> (نظام الاستجابة)
```typescript
{
  success: true, data: T, message?: string,
  request_id?: string, pagination?: PaginationMeta
}
// أو
{
  success: false, data: null,
  error: {
    code: string, message: string,
    field_errors?: Record<string, string[]>,
    request_id?: string
  },
  request_id?: string
}
```

## 5. المصادقة والصلاحيات

### AuthContext
- تخزين التوكنات في `localStorage` (مفاتيح: `madrasti_token`, `madrasti_refresh`)
- تسجيل الدخول: POST `/auth/login` → تخزين التوكنات → جلب `/me`
- تسجيل الخروج: POST `/auth/logout` → مسح التوكنات
- تحديث التوكن: استدعاء `/auth/refresh` عند 401 (مع منع التكرار)

### SchoolContext
- جلب قائمة المدارس: GET `/me/schools`
- تبديل المدرسة: POST `/me/switch-school`

### Middleware (حماية المسارات)
- **المسارات المحمية**:
  - `/admin/*`: يحتاج PRINCIPAL, DEPUTY_PRINCIPAL
  - `/admin/users`: PRINCIPAL, DEPUTY_PRINCIPAL فقط
  - `/admin/reports`: PRINCIPAL, DEPUTY_PRINCIPAL, ACADEMIC_SUPERVISOR
  - `/admin/attendance`: PRINCIPAL, DEPUTY_PRINCIPAL, ATTENDANCE_OFFICER, TEACHER
  - `/admin/correspondence`: PRINCIPAL, DEPUTY_PRINCIPAL فقط
  - `/admin/halaqat`: PRINCIPAL, DEPUTY_PRINCIPAL, ACADEMIC_SUPERVISOR, TEACHER
  - `/admin/points`: PRINCIPAL, DEPUTY_PRINCIPAL, TEACHER
  - `/admin/settings`: PRINCIPAL فقط
- **المسارات العامة**: `/login`, `/forgot-password`, `/reset-password`

### useEffectivePermissions (محاكاة RBAC عميل)
- خريطة صلاحيات ثابتة لـ 5 أدوار (PRINCIPAL, DEPUTY_PRINCIPAL, ACADEMIC_SUPERVISOR, TEACHER, ATTENDANCE_OFFICER)
- دالة `can(resource, action)` للتحقق من الصلاحية
- ملاحظة: هذا غير آمن أمنياً — الخادم هو من يطبق الصلاحيات فعلياً
